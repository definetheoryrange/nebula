# ResourceHub

ResourceHub 是一个面向开发人员与技术研究者的高质量技术资源导航与外链聚合平台。项目定位为技术信息的中转枢纽，通过人工筛选与自动化检测相结合的方式，收集并整理互联网中具有长期参考价值的技术文档、工具链入口、学术镜像站及社区讨论区。目标用户包括后端工程师、运维人员、技术写作译者以及计算机科学相关方向的研究生。项目致力于解决技术信息碎片化严重、优质入口难以追溯、个人书签管理成本过高等实际问题，将分散于网络各处的关键资源按主题域与内容类型进行系统化归集，并提供简洁的访问指引。

## 功能概览

- 外链资源结构化归档：将采集到的 URL 按主题域（如文化研究、语言技术、历史文献等）进行分类，支持标签过滤与模糊检索。

- 可用性健康检查：每日定时对收录资源进行 HTTP 状态码探测，自动标记异常链接并在前端面板中高亮提示。

- 自定义收藏夹导入：用户可通过上传 JSON 或 HTML 书签文件，将个人现有收藏批量合并至 ResourceHub 工作区。

- 资源变更历史追踪：记录每条外链的域名变更、证书状态变动及页面标题更新，支持时间线回溯。

- 短评与协作标注：注册用户可为任意资源添加技术性短评（如“该站提供 EPUB 格式出口”、“需使用特定 User-Agent 访问”），所有标注经审核后对全体可见。

- API 查询接口：提供 RESTful API，支持按域名、关键词、分类编码检索资源，便于二次开发集成。

- 黑暗模式与阅读版式：前端界面自动适配系统主题，且为长文档类资源提供无侧栏沉浸阅读模式。

## 应用场景

- 技术文档译者搜集术语平行语料：译者可通过 ResourceHub 快速定位多个中文技术社区与文档镜像站，交叉验证术语译法，提高翻译一致性。

- 运维人员监控依赖资源可用性：团队运维可将 ResourceHub 作为统一入口，利用内置健康检查模块观察第三方文档站与工具下载源的连通状态。

- 历史与人文方向研究者整理网络史料：研究者可借助分类归档功能，将不同来源的电子文献、影像资料入口按主题、年代、语种归集，形成个人研究门户。

- 新人快速了解领域常用入口：刚进入相关领域的工程师或学生可通过浏览资源分类树，在数分钟内获取该领域常用技术社区、代码托管镜像及学习路径参考。

- 技术写作团队维护对外参考列表：技术博客或开源项目文档组可使用 ResourceHub 管理其 README 中的“参考资料”区块，确保链接长期有效且版本可追溯。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL2 环境，需提前安装 Git 与 Node.js（>=18.x）。

```bash
# 克隆项目仓库
git clone https://github.com/resource-hub/resourcehub.git
cd resourcehub

# 安装依赖（使用 npm）
npm install

# 初始化本地资源索引与数据库结构
npm run init:db

# 启动开发服务器（默认监听 3000 端口）
npm run dev
```

访问控制台输出的本地地址（通常为 http://localhost:3000）即可进入资源浏览界面。首次启动会自动载入预置的 500 余条基础资源记录。

## 贡献指南

1. 查阅项目看板与议题列表：访问 GitHub Issues 页面，查找标记为 `help-wanted` 或 `good-first-issue` 的未解决问题，确认无重复工作后评论认领。

2. 派生仓库并创建功能分支：将主仓库派生至个人账户，随后克隆派生仓库至本地，基于 `main` 分支创建新分支，分支命名采用 `feature/功能简述` 或 `fix/问题简述` 格式。

3. 编写代码并补充单元测试：在 `backend/tests` 或 `frontend/tests` 对应目录下新增或更新测试用例，确保所有测试通过，且新增功能有相应测试覆盖。

4. 更新文档与资源列表：若新增或修改资源分类逻辑，需同步更新 `docs/` 目录下相关文档以及 `data/seeds/` 中的种子数据示例。

5. 发起拉取请求并等待审核：推送分支至派生仓库后，向主仓库的 `main` 分支发起 Pull Request，描述变更内容、影响范围及测试结果。审核通过后由维护者合并。

## 常见问题

问：健康检查模块显示部分资源为“异常”，但实际浏览器中可正常访问，如何调整探测参数？

答：默认健康检查使用 HEAD 请求且超时时间为 5 秒，部分站点可能对 HEAD 请求返回 405 或响应较慢。您可在 `backend/src/services/health.service.js` 中调整 `timeout` 值，或将请求方法改为 GET。对于已知特殊站点，建议在 `config/ignored-rules.json` 中单独配置跳过策略。

问：导入个人书签时提示“解析失败”，可能是什么原因？

答：ResourceHub 目前支持标准 Netscape 格式的 HTML 书签导出文件。若提示解析失败，请检查文件是否包含 `<DL>` 与 `<DT>` 标签结构。某些浏览器的导出文件头部可能缺少 `DOCTYPE` 声明，可尝试使用文本编辑器补全。若问题持续，建议使用 `scripts/import-bookmarks.js` 中的调试模式运行，查看具体行号异常。
