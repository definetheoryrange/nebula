# OpenBet Resource Hub

OpenBet Resource Hub 是一个面向体育数据爱好者和竞彩技术研究者的高质量外链与工具资源聚合平台。本项目不提供任何投注、预测或博彩服务，仅作为技术研究、数据可视化和统计分析的参考信息入口，旨在帮助研究人员、开发者和数据分析师快速获取结构化的体育赛事数据源、实时比分接口及历史统计资料。

项目定位为技术资源导航站，所有外链均经过基础可用性校验，并按数据类别进行组织，便于用户按需检索。适用于构建个人数据看板、赛事监控系统或学术研究中的数据集采集环节。

## 功能概览

- **实时比分数据源聚合** 汇聚多个亚洲地区体育数据服务商的实时比分页面，覆盖足球、篮球等主流赛事。

- **赛前预测与统计参考** 整理基于历史数据的赛前分析类网站，供算法模型训练和回测使用。

- **当日推荐与趋势索引** 提供按日期聚合的赛事推荐信息源，便于快速定位当日焦点赛事。

- **全场数据与分项统计** 区分全场比分与阶段比分数据源，满足不同粒度的数据分析需求。

- **历史版本数据回溯** 收录历史改版或旧版数据页面，用于对比数据字段变更和接口演进。

- **多语言与多时区适配** 所收录资源均支持多语言切换或时区显示调整，便于国际化开发测试。

- **外链健康度监控** 内置定时检测机制，对收录 URL 进行可达性和响应时间巡检，并在管理后台标注异常状态。

- **分类标签与全文检索** 每个资源可打多个标签，支持按赛事类型、数据频率、数据格式等进行快速过滤。

## 应用场景

- **个人数据看板开发** 开发者可利用本项目的资源列表，快速集成多个实时比分数据源，构建统一的赛事监控仪表盘。

- **预测模型训练数据采集** 数据科学家可借助历史比分和预测类网站，爬取结构化数据用于机器学习模型的训练与验证。

- **赛事信息聚合门户搭建** 技术团队可基于本项目提供的资源导航，搭建面向内部或公开的体育数据聚合门户，减少重复搜索成本。

- **学术研究与统计分析** 高校师生可在研究体育统计、观众行为分析等课题时，快速获取持续更新的数据来源。

- **接口兼容性测试** 质量保障人员可利用多版本数据页面，测试数据解析程序对不同 HTML 结构或 JSON 结构的兼容性。

## 快速开始

以下步骤将帮助您在本地环境快速运行本项目的资源导航站点。

```bash
# 1. 克隆代码仓库
git clone https://github.com/openbet-hub/resource-hub.git
cd resource-hub

# 2. 安装项目依赖（使用 npm）
npm install

# 3. 启动本地开发服务器
npm run dev
```

启动成功后，访问 `http://localhost:3000` 即可浏览资源列表。默认会加载内置的静态资源索引文件。

## 贡献指南

我们欢迎并鼓励社区贡献，包括新增资源链接、更新失效地址、优化分类逻辑和改进文档内容。所有贡献需遵循以下流程：

1. **查阅现有资源列表** 在提交新增或修改前，请先核对 `config/resources.json` 文件，确保目标 URL 尚未收录且类别不重复。

2. **提交 Issue 描述变更** 在 GitHub Issues 中创建一个新议题，明确说明新增资源或修改原因，附上原始 URL 和推荐分类。若为失效链接，请提供检测时间点和备选替代源。

3. **创建功能分支并编写修改** 从 `main` 分支切出 `feature/resource-update` 分支，按照 `resources.schema.json` 定义的格式编辑 JSON 文件，并同步更新本 README 中的资源列表章节。

4. **运行本地校验脚本** 执行 `npm run validate-resources` 检查所有 URL 的格式合法性和基础可达性，确保无重复条目和协议错误。

5. **提交 Pull Request** 推送到远程仓库后，创建 PR 至 `main` 分支，并在描述中关联对应 Issue。PR 需至少一名维护者审核通过后方可合并。

## 常见问题

**问：收录的某些链接无法访问，如何处理？**

答：本项目内置的外链健康检测会每 60 分钟自动探测一次所有资源 URL。若检测到连续三次不可达，管理后台会标记为「异常」状态。用户可在资源列表页面通过状态筛选查看异常链接，并提交 Issue 通知维护者复核。维护者确认失效后会将其移至归档列表，并在 5 个工作日内寻找替代数据源。

**问：我想在资源列表中添加一个非体育类的数据源，是否可以？**

答：当前项目定位严格限定为体育赛事相关的数据资源。若您推荐的数据源包含明确的赛事比分、统计或预测类信息，可在 Issue 中说明其体育关联性。若为纯金融、天气或新闻类数据，暂不纳入收录范围。但对于体育衍生领域（如场馆天气、球员社交热度），可酌情讨论。

**问：项目是否提供 API 接口供外部程序调用资源列表？**

答：是的。启动服务后，访问 `/api/resources` 可获得 JSON 格式的完整资源列表，包含分类、标签、最后校验时间和当前状态。同时支持 `?category=score` 和 `?status=active` 等查询参数进行过滤。API 响应遵循 JSON:API 规范，便于程序化集成。
