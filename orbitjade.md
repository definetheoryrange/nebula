# VastLink Navigator

VastLink Navigator 是一个面向技术内容聚合与资源导航的开源项目，旨在为开发者、技术研究者以及互联网信息采集人员提供一个轻量级、可扩展的外链资源管理与展示平台。该项目定位于中大型技术团队的内容中台辅助工具，帮助用户系统化地归集、分类和检索分布在不同域名下的结构化信息资产，有效降低信息碎片化带来的检索与管理成本。

与传统书签管理或简单的链接收藏工具不同，VastLink Navigator 内置了标签体系、状态监控与访问热度统计等基础能力，允许运维人员快速识别失效资源，并支持以 Markdown 或 JSON 格式导入/导出资源清单，便于集成至现有的 CI/CD 或文档生成流程中。项目本身不依赖任何商业服务，完全基于开源生态构建，适合私有化部署或二次开发。

## 功能概览

- **多维度资源分类与标签管理**：支持为每个外链资源分配多个自定义标签，并基于标签进行快速筛选与聚合展示，便于建立符合业务场景的知识体系。

- **链接状态健康检查**：内置异步巡检任务，可定期探测已收录资源的可访问性，并通过仪表盘高亮显示异常链接，辅助运维人员及时清理或更新。

- **访问热度统计与排序**：记录每个资源链接的被点击次数与最近访问时间，支持按热度、添加时间或字母顺序动态排序，帮助团队识别高频使用资源。

- **全文检索与模糊匹配**：针对资源标题、描述、标签及所属分类进行全文关键词检索，支持中文分词与拼音首字母快速定位，提升大型资源库的查找效率。

- **数据导入导出与备份机制**：提供 JSON 与 CSV 格式的批量导入导出接口，便于与外部数据源同步；同时内置自动备份策略，可定期将资源清单归档至指定存储目录。

- **响应式管理面板与暗色主题**：提供基于 Web 的管理界面，适配桌面与移动设备，并内置明暗两种配色方案，满足不同使用环境下的视觉偏好。

- **插件化扩展支持**：采用面向接口的插件设计，允许开发者通过编写简单 Python 脚本扩展自定义资源解析器或通知处理器，无需修改核心代码。

## 应用场景

- **技术团队内部文档资源中台**：研发团队可将常用的 API 文档、组件库地址、内部工具链入口以及运维手册等分散链接统一收录至 VastLink Navigator，通过标签区分开发、测试、生产环境，显著减少成员在多个平台间切换查找信息的时间。

- **开源项目外部依赖与参考资源整理**：开源项目维护者可使用本工具整理项目所依赖的上游仓库、参考论文、数据源地址以及社区讨论贴，并将资源列表嵌入项目文档或 README 中，方便贡献者快速了解项目的技术背景与外部依赖关系。

- **信息安全研究者的威胁情报源聚合**：安全研究人员可利用本工具的标签与健康检查功能，持续跟踪多个情报站点、漏洞数据库及厂商安全公告页面的可用性变化，一旦某情报源失效，系统即可发出提醒，避免依赖过期信息。

- **企业培训与新人 onboarding 知识导航**：企业培训部门可搭建内部导航站，将新员工入职所需接触的各类系统入口、学习课程链接、规章制度页面以及常用工具下载地址集中管理，配合热度统计功能了解新员工最常访问的资源，从而优化培训材料的编排顺序。

## 快速开始

以下步骤将指导您在本地环境快速启动 VastLink Navigator 开发实例。

```bash
# 1. 克隆项目仓库
git clone https://github.com/your-org/vastlink-navigator.git
cd vastlink-navigator

# 2. 安装项目依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 下请使用 venv\Scripts\activate
pip install -r requirements.txt

# 3. 初始化本地配置文件与数据库
cp .env.example .env
python scripts/init_db.py

# 4. 导入初始示例资源数据（可选）
python scripts/load_demo_data.py

# 5. 启动开发服务器
python app.py runserver --host=0.0.0.0 --port=8080
```

启动成功后，在浏览器中访问 `http://localhost:8080` 即可进入管理面板。默认管理员账号为 `admin`，密码为 `admin123`，首次登录后请及时修改。

## 常见问题

Q: 项目启动时报错 "sqlite3.OperationalError: no such table: links"，该如何处理？

A: 该问题通常是因为未正确执行数据库初始化脚本所致。请确认在启动服务之前已运行 `python scripts/init_db.py` 命令。如果数据库文件已存在但表结构缺失，可删除 `data/vastlink.db` 文件后重新执行初始化操作。同时请检查 `.env` 文件中的数据库连接字符串是否指向了正确的文件路径。

Q: 生产环境下如何启用 Redis 缓存支持以提升性能？

A: 首先确保 Redis 服务已安装并运行在可访问地址。然后在 `.env` 配置文件中将 `CACHE_BACKEND` 设置为 `redis`，并正确填写 `REDIS_HOST`、`REDIS_PORT` 及可选的 `REDIS_PASSWORD` 和 `REDIS_DB` 参数。重启服务后，系统将自动使用 Redis 作为缓存与临时数据存储后端。若未配置 Redis，系统将回退至本地内存缓存，该模式仅在单进程开发环境下可靠。

Q: 如何将现有的浏览器书签或 CSV 文件批量导入至系统中？

A: 您可以使用项目提供的导入脚本，或通过管理界面中的「导入数据」功能。具体步骤为：准备 CSV 文件，列标题需包含 `title`、`url`、`description` 和可选的 `tags`（多个标签以竖线分隔）。然后在管理界面点击「导入」并上传文件，系统将自动解析并去重添加。若数据量极大（超过 1 万行），推荐使用命令行脚本 `python scripts/import_from_csv.py --file 您的文件.csv`，该脚本支持分批提交与进度显示。
