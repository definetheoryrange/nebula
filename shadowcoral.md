# ResourceHub

ResourceHub 是一个面向技术开发者与内容研究者的高质量外链与资源导航系统。项目定位为结构化网络资源聚合平台，专注于收录、分类与索引各类具备明确主题指向的垂直领域信息入口。目标用户包括爬虫工程师、数据分析从业者、内容分类研究人员以及网络资源管理专员。ResourceHub 解决的核心问题在于：网络资源分散、入口易失效、分类标准不统一、检索路径不透明，导致信息获取成本高且可复用性差。通过建立标准化的外链清单与主题映射机制，项目致力于成为技术团队内部资源管理的基础设施组件。

## 功能概览

- 垂直领域外链聚合：按主题类别对网络资源入口进行结构化整理，每个链接附带主题标签与入库时间戳，支持快速筛选。
- 资源状态健康检查：内置链接可访问性探测机制，定期对收录 URL 进行连通性验证，自动标记异常条目并生成告警日志。
- 分类目录树浏览：采用多级分类目录对资源进行组织，支持按类别展开与折叠，便于快速定位特定主题下的所有链接。
- 全文检索与过滤：提供基于关键词的标题与描述检索能力，同时支持按类别、状态、更新时间等多维度过滤。
- 批量导入与导出：支持通过 CSV 与 JSON 格式批量导入外链数据，并支持将当前资源列表完整导出为标准化格式。
- 访问统计与热度排序：记录每个资源入口的点击次数与最近访问时间，支持按热度排序，辅助识别高频使用资源。
- 用户自定义标签系统：允许用户为任意资源添加自定义标签，实现个性化分类，标签数据支持本地持久化。
- 资源变更订阅：通过 RSS 或 Webhook 方式对外推送资源新增、删除与状态变更事件，便于下游系统实时同步。

## 应用场景

- 数据采集工程师构建定向爬虫种子文件：在启动大规模爬取任务前，需要准备一批高质量的初始 URL 作为爬虫入口。ResourceHub 提供按主题归类的稳定链接集合，可快速导出为种子列表，显著降低入口收集与清洗成本。
- 内容分类研究人员建立训练数据集标签映射：在进行文本分类或主题建模时，需要大量带有明确主题标注的网页样本。ResourceHub 中每个链接均关联主题标签，可直接用于构建正负样本集合，辅助模型训练与验证。
- 网络运维人员监控第三方资源可用性：企业级应用往往依赖大量第三方外部服务或数据源。运维团队可利用 ResourceHub 的健康检查功能，定期探测依赖资源的状态，在源站异常时及时触发告警与切换策略。
- 技术团队内部知识库资源沉淀：研发团队在日常工作中积累了大量有用外链，但缺乏统一管理入口。ResourceHub 可作为内部知识库的补充模块，提供标准化的资源入库、分类与共享机制，减少重复查找时间。
- 合规审查团队进行内容入口备案与审计：法务或合规部门需要对团队使用的外部资源进行备案与定期审查。ResourceHub 提供完整的资源列表导出与变更记录，支持按时间范围回溯历史状态，满足内控与审计要求。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceHub 服务。

```bash
# 克隆项目仓库
git clone https://github.com/resourcehub/resourcehub.git

# 进入项目目录
cd resourcehub

# 安装项目依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 初始化本地数据库
python scripts/init_db.py

# 启动开发服务器
python app.py runserver --host 0.0.0.0 --port 8080
```

服务启动后，访问 http://localhost:8080 即可进入 ResourceHub 管理界面。默认管理员账号为 admin，初始密码请在首次启动时查看控制台输出日志获取。

## 安装要求

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.10 及以上 | 核心运行环境，低于此版本将导致异步语法兼容性问题 |
| SQLite | 3.35 及以上 | 默认内置数据库，用于存储资源条目、标签与访问日志 |
| Redis | 6.0 及以上 | 可选依赖，用于缓存与分布式任务队列，生产环境建议安装 |
| Node.js | 16.0 及以上 | 仅用于前端静态资源构建，后端运行时无需此环境 |
| pip | 22.0 及以上 | Python 包管理器，用于安装 requirements.txt 中声明的第三方库 |
| 网络连通性 | 出方向 80/443 端口开放 | 健康检查模块需要对外发起 HTTP/HTTPS 请求，需确保网络策略允许 |

## 常见问题

Q: 启动服务时提示数据库版本不兼容，如何解决？

A: 请先确认本地 SQLite 版本是否不低于 3.35。您可以通过 `sqlite3 --version` 命令检查。若版本过低，请升级 SQLite 或使用 Docker 镜像运行（Dockerfile 中已内置合适版本）。如果使用生产级数据库（如 PostgreSQL），请修改配置文件中的数据库连接字符串并执行迁移脚本。

Q: 健康检查模块报告大量链接超时，但浏览器可以正常访问，是什么原因？

A: 健康检查默认使用 requests 库的默认超时设置（连接超时 5 秒，读取超时 10 秒）。部分源站响应较慢或存在防火墙策略，可能拒绝来自非浏览器 User-Agent 的请求。您可以在配置文件中调整超时阈值，或启用 `--browser-mode` 参数模拟真实浏览器请求头。另外，请确认运行环境出方向网络策略是否允许访问目标 IP 与端口。

Q: 如何将 ResourceHub 部署到生产环境并保持数据持久化？

A: 建议使用 Docker Compose 进行生产部署。项目根目录提供了 docker-compose.yml 示例文件，其中包含应用容器与 Redis 容器定义。您需要将 SQLite 数据文件挂载到宿主机持久化目录，或切换为 PostgreSQL 作为后端数据库。同时，生产环境应配置环境变量覆盖默认密钥、调试模式与日志级别。具体步骤请参考 /docs/ops-guide/production-deployment.md。
