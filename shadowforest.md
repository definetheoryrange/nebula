# ResourceHub

ResourceHub 是一个面向技术社区与内容创作者的轻量级外链资源导航与信息汇总系统。项目定位为结构化技术资源的集散地，帮助开发者、研究员与内容运营人员快速定位分散于互联网各处的优质外部资源，解决信息碎片化导致的检索效率低下与资源可信度难以评估的问题。系统采用静态站点生成方式，以最低运维成本实现高可读性、高组织性的资源展示层，同时内置外链健康检查与访问状态标记功能，适用于个人知识库、团队文档站或社区共享入口等场景。ResourceHub 不存储任何外部文件，仅提供元数据与超链接，遵循互联网资源共享的公开原则。

## 功能概览

- **多级分类索引**：支持按主题、地域、语种或文件类型对资源进行标签化分类，每个分类下可无限嵌套子目录，便于精细化管理。
- **外链状态监测**：每日定时检测收录资源的 HTTP 状态码，自动标记异常链接（超时、404、SSL 证书错误等），并在前端界面高亮提示。
- **自定义元数据模板**：允许为每条资源附加自定义字段（如收录日期、维护人、更新频率、备用地址），满足企业级文档管理需求。
- **全文检索与过滤**：集成轻量级倒排索引，支持按标题、描述、分类、标签进行关键词搜索，并支持多条件组合过滤。
- **静态导出与部署**：支持一键生成纯静态 HTML 文件，可部署于任意 Web 服务器或对象存储服务，无需数据库或后端运行时。
- **资源导入与导出**：提供 CSV 与 JSON 格式的批量导入导出接口，便于与其他系统（如书签管理器、知识库）进行数据迁移。
- **访问统计看板**：内置基于页面浏览量（PV）与点击量的简易统计模块，帮助运营者了解热门资源与用户关注趋势。
- **响应式布局**：适配桌面、平板与移动设备，确保在不同屏幕尺寸下均获得良好的浏览与操作体验。

## 应用场景

- **技术团队内部文档中心**：研发团队可使用 ResourceHub 汇总常用的 API 文档、开源库主页、运维手册与内部工具地址，替代浏览器书签的分散管理方式，新成员入职时可快速获取所有必要外部入口。
- **开源社区贡献者入口**：开源项目维护者可利用 ResourceHub 构建贡献者导航页，集中放置代码仓库地址、贡献指南、讨论区、CI/CD 状态面板和版本发布记录，降低参与门槛。
- **在线课程辅助资源库**：教育机构或独立讲师可为每门课程配套搭建 ResourceHub 实例，将参考论文、视频链接、在线编译器和习题集锦按课时分类，学生无需自行搜索即可直达权威材料。
- **垂直领域信息聚合站**：针对特定行业（如生物信息、金融量化、前端工程）运营者，可每日更新领域内的高质量博客、新闻源与工具站点，为从业者提供一站式信息入口。

## 快速开始

以下步骤将指导您在本地环境中完成 ResourceHub 的克隆、依赖安装与开发服务器启动。

```bash
# 1. 克隆项目仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 2. 安装项目依赖（使用 npm）
npm install

# 3. 启动开发服务器（默认监听端口 3000）
npm run dev
```

执行上述命令后，在浏览器中访问 `http://localhost:3000` 即可预览站点。生产环境构建请使用 `npm run build` 配合 `npm run start`。

## 项目结构

项目采用模块化单体架构，核心源码集中于 `src/` 目录，配置与构建脚本位于根目录。以下为关键目录与文件说明。

```
resourcehub/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心业务逻辑（资源管理、分类树、索引引擎）
│   ├── server/                    # Express 服务端代码（路由、中间件、API 控制器）
│   ├── client/                    # 前端静态资源（React 组件、样式表、图片字体）
│   │   ├── components/            # 可复用 UI 组件（导航卡、资源表格、状态徽章）
│   │   ├── pages/                 # 页面级组件（首页、分类视图、详情页、看板）
│   │   └── hooks/                 # 自定义 React Hooks（数据请求、本地存储、防抖）
│   ├── lib/                       # 通用工具函数（日期格式化、URL 解析、状态码映射）
│   ├── workers/                   # 后台任务线程（外链健康检查、统计聚合）
│   └── config/                    # 环境配置与常量定义（分类预设、默认标签）
├── tests/                         # 单元测试与集成测试（Jest + Supertest）
├── docs/                          # 完整项目文档（含用户手册、运维指南、API 参考）
├── scripts/                       # 构建与运维辅助脚本（数据迁移、种子数据生成）
├── public/                        # 静态资源输出目录（构建后自动生成）
├── .env.example                   # 环境变量模板（数据库连接、监控开关）
├── package.json                   # npm 清单（依赖声明、脚本命令）
├── tsconfig.json                  # TypeScript 编译配置
├── webpack.config.js              # 前端构建配置（开发与生产模式）
└── README.md                      # 项目入口说明文档（即本文档）
```

## 常见问题

**问：ResourceHub 是否依赖数据库？如何存储资源数据？**

答：ResourceHub 默认不使用关系型数据库或 NoSQL 系统。所有资源数据以 JSON 文件形式存储在项目根目录的 `/data` 文件夹下，便于版本控制、手动编辑和快速迁移。生产环境下，这些 JSON 文件会在应用启动时加载至内存，并定期写入备份。如需使用外部数据库，官方提供了可选的 PostgreSQL 适配器扩展包，但不属于核心依赖。

**问：外链健康检查功能是否会拖慢页面访问速度？**

答：健康检查任务由独立的 Worker 线程在后台定时执行（默认间隔 12 小时），完全异步且不阻塞主请求链路。前端渲染时直接从内存缓存中读取检测结果状态码，时间复杂度为 O(1)。即使某个外部资源响应超时，也不会影响 ResourceHub 自身页面的加载性能。您可以在配置文件中调整检测间隔或完全关闭该功能。

**问：如何将现有的浏览器书签批量导入到 ResourceHub？**

答：ResourceHub 管理后台的“导入”功能区支持解析 Netscape 书签格式（HTML）以及主流浏览器的 CSV 导出文件。您只需从浏览器导出书签，然后在管理界面上传文件，系统会自动识别文件夹结构并转换为分类与资源条目。对于不支持的格式，我们提供了一份 Python 转换脚本，位于 `/scripts/import-tools/` 目录下，可供高级用户自定义映射规则。
