# ResourceHub

一个面向开发人员与技术决策者的高质量技术资源导航与外部链接汇总站。ResourceHub 致力于解决技术信息过载与优质内容分散的问题，通过人工筛选与社区推荐机制，将分散在社区、官方文档、技术博客、互动教程等不同载体的高质量技术资源进行结构化整理，并以清晰、可检索的方式呈现给用户。项目主要面向中高级开发者、技术团队负责人、开源贡献者以及正在学习系统设计、编程语言、运维与工程实践的开发者群体。

ResourceHub 并非一个简单的书签集合，而是一个具备基础内容编排、标签分类、全文检索与快照存档能力的轻量级知识管理工具。项目后端基于 Node.js 与 SQLite 构建，前端采用响应式设计，支持在桌面与移动端无障碍访问。项目所有外链数据均通过社区贡献与自动化巡检维护，确保链接有效性与内容时效性。

## 功能概览

- **精选资源导航**：按技术领域、难度级别与应用场景对资源进行三级分类，支持快速筛选与定位。
- **全文检索与标签过滤**：内置倒排索引与标签系统，支持对资源标题、描述、来源域名进行多维度检索。
- **社区推荐与投票机制**：注册用户可提交新资源链接，其他用户可投票赞同或反对，高质量链接自动提升排名。
- **链接健康状态监控**：每日定时检测已收录外链的 HTTP 状态码与 DNS 解析情况，自动标记失效或重定向链接。
- **个人收藏与自定义集合**：用户可将常用资源加入个人收藏夹，并创建自定义分组便于项目复用。
- **开放数据导出与 API 接口**：支持将资源列表导出为 JSON、CSV 或 OPML 格式，并提供 RESTful API 供第三方工具调用。
- **快照存档与阅读模式**：对部分长文类资源提供静态快照与清爽阅读模式，减少广告干扰，提升阅读体验。

## 应用场景

- **团队技术选型调研**：当团队需要评估新的前端框架、数据库中间件或云原生工具时，可通过 ResourceHub 快速查找官方文档、社区评测、性能对比与实战案例链接，减少重复搜索与信息筛选成本。
- **个人系统化学习路径规划**：开发者可围绕某一主题，如「分布式系统」「Rust 编程」「Kubernetes 运维」，在平台上查找从入门到进阶的教程、视频课程与开源项目镜像，构建体系化学习路线。
- **开源项目维护者文档引用**：开源项目维护者可将 ResourceHub 作为项目 README 或官方文档的外部引用来源，集中列出依赖工具、参考文档与社区讨论链接，降低用户阅读门槛。
- **技术会议与沙龙资料归档**：组织者可在会议结束后，将讲稿、视频回放、相关代码仓库与延伸阅读链接统一收录至 ResourceHub 专题页面，便于与会者后续查阅与二次传播。
- **个人知识库外链补充**：使用 Notion、Obsidian 或 Logseq 等工具的知识管理爱好者，可通过导出功能将 ResourceHub 中的精选链接批量导入个人知识库，丰富外脑资源池。

## 快速开始

以下步骤适用于开发环境部署与本地测试。生产环境部署请参考文档导航章节中的部署指南。

```bash
# 克隆代码仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 安装项目依赖（使用 npm）
npm install

# 初始化 SQLite 数据库与默认分类数据
npm run init-db

# 启动开发服务器（默认监听 3000 端口）
npm run dev
```

启动成功后，在浏览器中访问 <code>http://localhost:3000</code> 即可浏览本地实例。首次启动会自动创建管理员账户，控制台会打印初始密码，请及时修改。

## 常见问题

**Q：我提交的资源链接为什么没有立即出现在公开列表中？**

A：为保障资源质量与安全性，所有用户提交的新链接均需经过人工审核与自动安全检查。审核周期通常为 1 至 3 个工作日。您可在个人中心「我的提交」页面查看审核状态。若链接被拒绝，系统会附带拒绝原因说明，您可根据说明修改后重新提交。

**Q：项目是否支持完全离线部署？**

A：支持。ResourceHub 不依赖任何外部在线服务（如 CDN、第三方 API 或云数据库）。所有静态资源与数据库均内嵌于项目包中。您只需将代码完整复制到内网服务器，并确保 Node.js 环境可用，即可实现完全离线运行。快照功能会在首次访问外链时自动缓存页面内容，后续访问无需再次请求外部站点。

**Q：如何迁移或备份我的资源数据？**

A：您可以通过管理员后台或命令行脚本 <code>npm run export</code> 将当前资源列表、分类标签与用户收藏数据导出为 JSON 或 CSV 格式。完整数据迁移需要同时备份 <code>data/</code> 目录下的 SQLite 数据库文件以及 <code>config/</code> 目录中的自定义配置。导入数据时，请使用 <code>npm run import -- --file=xxx.json</code> 命令，具体参数请参考部署文档中的数据迁移章节。
