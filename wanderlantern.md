# LinkHub Analytics

LinkHub Analytics 是一个面向数据驱动型开发者与运营人员的技术资源外链聚合平台，专注于采集、清洗与分类展示互联网优质技术内容、数据看板与实时分析工具。项目定位为“技术导航的元索引”，帮助用户从海量信息中快速定位可用数据源、监控面板与行业动态页面，减少重复搜索成本，提升信息获取效率。

项目默认以只读静态站点方式运行，核心数据基于 Markdown 与 JSON 结构化存储，支持一键克隆、本地预览与自动化部署。所有外链资源均经过类别标记与可用性校验，适合个人开发者、数据分析团队、技术内容运营者以及独立站站长作为日常信息中台使用。

## 功能概览

- **外链分类看板**：按行业、地域、数据类型等多维度对收录链接进行分组展示，支持快速筛选与模糊搜索。
- **可用性轮询检测**：内置轻量级 HTTP 状态检查脚本，可定时探测各资源域名响应状态，辅助判断数据源稳定性。
- **标签化收藏系统**：支持为每个链接添加自定义标签，便于构建个人化的资源集合，并支持标签云导航。
- **本地化全文检索**：基于静态 JSON 索引实现标题与描述的全文匹配，无需第三方搜索服务，适合内网部署。
- **响应式数据预览**：对部分公开 API 类链接提供简单的 JSON 响应预览功能，便于快速验证数据格式。
- **变更历史记录**：每次更新资源列表时自动生成 CHANGELOG 片段，记录增删改操作，便于团队协作追溯。
- **一键导出报告**：支持将当前筛选后的链接列表导出为 CSV 或 Markdown 表格，用于周报或知识库归档。
- **暗色主题与阅读模式**：内置两套视觉主题，并针对长文内容提供专注阅读布局，减少视觉干扰。

## 应用场景

- **技术团队内部知识库导航**：将团队常用的监控面板、日志查询入口、API 文档站集中管理，新成员入职时可一键获取所有关键资源，减少交接沟通成本。
- **数据运营日报制作**：运营人员可每日访问本平台，快速打开竞品数据站点、行业趋势页面以及自有后台，结合导出报告功能生成当日数据快照。
- **开源项目外部依赖索引**：当开源项目需要引用多个外部数据源或参考站点时，可使用本平台统一维护依赖链接列表，并在项目 README 中引用，提升文档可维护性。
- **个人技术信息聚合**：独立开发者或研究员可将个人常读的技术博客、论文预印本站点、在线编译器工具集中管理，打造私人的信息起始页。

## 快速开始

以下步骤帮助您在本地环境快速启动 LinkHub Analytics 实例。

```bash
# 1. 克隆项目仓库
git clone https://github.com/linkhub-analytics/linkhub-core.git
cd linkhub-core

# 2. 安装依赖（基于 Node.js 18+ 与 npm）
npm install

# 3. 构建静态资源并启动开发服务器
npm run build
npm run serve
```

启动成功后，访问控制台输出的本地地址（默认为 http://localhost:8080）即可进入首页。首次启动会自动生成示例数据，您可以直接浏览或替换为自己的链接配置。

## 项目结构

项目采用模块化目录组织，核心代码与配置分离，便于维护与二次开发。

```
linkhub-core/
├── .github/                     # GitHub 工作流配置（CI / 自动化检测）
├── build/                       # 构建脚本与 Webpack 配置
│   ├── webpack.common.js        # 基础构建配置
│   ├── webpack.dev.js           # 开发环境配置
│   └── webpack.prod.js          # 生产环境优化配置
├── public/                      # 静态资源入口（不经过构建）
│   ├── favicon.ico              # 站点图标
│   └── robots.txt               # 搜索引擎爬虫指引
├── src/                         # 源代码主目录
│   ├── assets/                  # 图片、字体等静态资源
│   ├── components/              # Vue / React 风格 UI 组件
│   ├── hooks/                   # 自定义逻辑钩子（状态管理、请求）
│   ├── pages/                   # 路由页面组件（首页、分类、详情）
│   ├── services/                # API 调用与数据解析服务
│   ├── store/                   # 全局状态管理（Pinia / Redux）
│   ├── styles/                  # 全局样式与主题变量（SCSS）
│   ├── utils/                   # 工具函数（格式化、校验、日期处理）
│   └── index.js                 # 前端应用入口
├── data/                        # 链接资源数据存储（JSON / YAML）
│   ├── categories.json          # 分类定义
│   ├── links.json               # 主链接列表（含标签、状态）
│   └── changelog.json           # 变更历史
├── scripts/                     # 运维与辅助脚本
│   ├── health-check.js          # 外链可用性轮询脚本
│   ├── generate-sitemap.js      # 站点地图自动生成
│   └── import-csv.js            # 从 CSV 批量导入链接
├── tests/                       # 单元测试与集成测试
│   ├── unit/                    # 组件与工具函数单测
│   └── e2e/                     # 端到端场景测试（Playwright）
├── docs/                        # 项目文档（用户手册与开发者指南）
├── .env.example                 # 环境变量示例文件
├── .gitignore                   # Git 忽略规则
├── package.json                 # 项目依赖与脚本定义
├── README.md                    # 项目说明（本文件）
└── LICENSE                      # MIT 许可证文本
```

## 贡献指南

我们欢迎各类形式的贡献，包括但不限于新增链接资源、修复 UI 显示问题、优化构建流程或完善文档。

1. **提交 Issue 讨论**：在提交 Pull Request 前，请先在 Issues 区域提出您的修改意图或发现的问题，避免重复工作。建议使用提供的模板描述问题背景或功能需求。
2. **Fork 仓库并创建特性分支**：从主仓库 Fork 后，请基于 `main` 分支创建 `feature/xxx` 或 `fix/xxx` 格式的分支，并保持分支名称语义清晰。
3. **遵循代码规范与测试要求**：提交前执行 `npm run lint` 与 `npm run test` 确保代码风格统一且不破坏现有功能。新功能需附带对应单元测试。
4. **更新资源数据或文档**：若贡献涉及链接增删，需同步更新 `/data/links.json` 以及相关分类文件，同时补充 `changelog.json` 中的记录。文档修改需与代码变更保持一致。
5. **发起 Pull Request**：通过 GitHub 页面发起 PR，并在描述中关联对应的 Issue 编号。PR 需通过 CI 检查（包含构建与测试环节）后，由维护者进行代码审查。

## 常见问题

**Q：平台是否支持在线编辑链接数据，还是必须通过本地文件修改？**  
A：当前版本以静态数据为主，所有链接修改需要通过本地编辑 `/data/` 目录下的 JSON 文件并重新构建。我们计划在未来版本中增加管理后台界面，支持在线 CRUD 操作，但现有架构优先保证数据可追溯性与版本控制能力。

**Q：外链可用性检测会影响平台自身访问速度吗？**  
A：检测脚本独立于前端服务运行，建议通过定时任务（如 cron）在后台异步执行，并将结果缓存至本地 JSON 文件中。前端仅读取缓存状态，不会阻塞页面渲染。默认配置下，检测超时设置为 5 秒，并自动忽略非关键资源以降低影响。

**Q：如何迁移已有的大量书签到本平台？**  
A：我们提供了 `scripts/import-csv.js` 辅助脚本，支持从标准 CSV 格式（列名：标题、URL、分类、标签、描述）导入数据。您可以从浏览器书签导出工具或 Notion 表格生成符合格式的 CSV 文件，然后运行导入命令即可自动合并至 `links.json`。
